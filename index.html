Gérer les comptes. Les mots de passe sont dérivés via PBKDF2-SHA256 (100 000 itérations) avec sel unique par compte avant stockage local — jamais en clair.

⚠ Limite de sécurité : ces comptes (identifiants + hashs de mots de passe + code de secours) sont stockés dans le navigateur (localStorage) et inclus dans la synchronisation, pour permettre à un nouvel appareil de les récupérer via "Première connexion / Configurer l'accès". Seuls des hashs PBKDF2 (jamais les mots de passe en clair) circulent ainsi — mais toute personne ayant accès à votre Gist/JSONBin/serveur de synchronisation peut donc voir la liste des comptes et tenter un cassage hors-ligne des mots de passe. Choisissez des mots de passe robustes et gardez l'accès à votre source de synchronisation restreint aux membres de confiance.

☁ Synchronisation partagée

Stocker les données sur un service partagé, accessible à tous les membres du conseil syndical. La configuration ci-dessous est sauvegardée dans votre navigateur.

💡 Pour que tout le monde voie les mêmes données : configurez la connexion ci-dessous, testez-la, puis cliquez sur "Activer". Partagez ensuite le lien de l'application (sans configuration intégrée) à tous les membres. À leur première ouverture, ils cliquent sur "Première connexion / Configurer l'accès" sur l'écran de connexion, saisissent les mêmes informations de connexion que ci-dessous (token/clé), et récupèrent ainsi automatiquement les comptes existants et les données du conseil syndical.

⚠️ Évitez "📦 Générer une copie configurée" si vous comptez héberger le fichier sur un espace public (ex: GitHub Pages) : il intègre votre clé/token dans le code, ce que GitHub bloque (protection des secrets) et qui exposerait cette clé à tous les visiteurs.
